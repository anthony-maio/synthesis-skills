# Code of Conduct

This project follows a simple standard: be direct, respectful, and review ideas rigorously without turning collaboration into theater.

## Expected Behavior

- critique code and proposals, not people
- provide provenance when adapting external work
- keep review comments concrete and actionable
- assume good faith, but require technical clarity

## Unacceptable Behavior

- harassment or personal attacks
- deceptive attribution
- intentionally low-signal spam or abusive PRs

Project maintainers may moderate discussions and contributions to keep the repository usable.
